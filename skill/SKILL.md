# Route Optimizer - 智能路线规划

规划最优驾车路线，支持明确途经点和模糊途经点（如"找个超市"）。

---

## 首次使用：配置 API Key ⚠️ 重要

**首次使用此 Skill 的用户需要配置高德地图 API Key。**

### 一、API Key 获取步骤（图文详解）

**步骤 1：访问高德开放平台**
```
打开浏览器，访问：https://console.amap.com
```

**步骤 2：注册/登录账号**
- 如果没有账号，点击"注册"按提示完成注册
- 使用手机号或邮箱注册，完成实名认证

**步骤 3：创建应用**
1. 登录后，在控制台左侧找到"应用管理" → "我的应用"
2. 点击"创建新应用"
3. 应用名称填写：`路线规划`（任意名称均可）
4. 点击"确定"

**步骤 4：添加 Key**
1. 在应用详情页面，点击"添加 Key"
2. Key 名称填写：`路线规划Key`
3. **服务平台选择：`Web服务`（重要！）**
4. 勾选"我已阅读并同意高德地图服务协议"
5. 点击"提交"

**步骤 5：获取 Key**
- 在 Key 列表中，找到刚才创建的 Key
- **复制那个看起来很长的字符串**（例如：`8e2f1234567890abcdef1234567890ab`）
- 这就是你的 API Key

### 二、把 API Key 给 AI

**直接告诉 AI 这句话：**
> "我的高德 API Key 是：XXXXX"（把刚才复制的 Key 粘贴进去）

或者 AI 会主动提示你输入。

### 三、验证配置成功

配置成功后，你可以测试：
> "帮我规划从南京站到新街口的路线"

## 自然语言参数提取规则 ⚠️ 关键

**当用户提出路线规划需求时，AI 必须从用户输入中提取以下参数：**

### 用户输入示例

> "我想要从仙鹤门地铁站到南京新百，途径汉堡王和鲜芋仙新街口店"

### 提取规则

| 参数 | 提取方式 | 示例结果 |
|------|---------|---------|
| **origin（起点）** | 从"从X到Y"或"X到Y"中提取起点 | `仙鹤门地铁站` |
| **destination（终点）** | 从"从X到Y"或"X到Y"中提取终点 | `南京新百` |
| **waypoints（途经点）** | 从"途径X"或"途经X"或"经过X"中提取 | 见下方 |

### 途经点类型判断

**明确途经点**：用户给出了具体的店名/地址
```
途径汉堡王 → {"type": "explicit", "name": "汉堡王"}
途径鲜芋仙新街口店 → {"type": "explicit", "name": "鲜芋仙新街口店"}
```

**模糊途经点**：用户只给了类型关键词
```
途径超市 → {"type": "fuzzy", "keyword": "超市"}
找个加油站 → {"type": "fuzzy", "keyword": "加油站"}
途径餐厅 → {"type": "fuzzy", "keyword": "餐厅"}
```

### 完整参数示例

用户输入：
> "我想要从仙鹤门地铁站到南京新百，途径汉堡王和鲜芋仙新街口店"

提取后的参数：
```json
{
  "action": "optimize",
  "origin": "仙鹤门地铁站",
  "destination": "南京新百",
  "waypoints": [
    {"type": "explicit", "name": "汉堡王"},
    {"type": "explicit", "name": "鲜芋仙新街口店"}
  ]
}
```

用户输入：
> "从南京站到新街口，路上找个超市买点东西"

提取后的参数：
```json
{
  "action": "optimize",
  "origin": "南京站",
  "destination": "新街口",
  "waypoints": [
    {"type": "fuzzy", "keyword": "超市"}
  ]
}
```

## 调用方式

提取参数后，执行以下命令：

```bash
python D:/code/route-optimizer/skill/scripts/run.py '<参数JSON>'
```

或者通过 Python 代码调用：

```python
from core import optimize_route

result = optimize_route(
    origin="仙鹤门地铁站",
    destination="南京新百",
    waypoints=[
        {"type": "explicit", "name": "汉堡王"},
        {"type": "explicit", "name": "鲜芋仙新街口店"}
    ]
)
```

## 注意事项

- 途经点总数不能超过 8 个
- 模糊途经点会在路线中点附近 5km 范围内搜索
- 建议使用具体的地址或 POI 名称以提高准确性
- 需要 Python 3.8+ 和网络连接
- API Key 免费额度：个人开发者每日 5000 次调用，足够日常使用

---

## 技术实现

- 使用高德地图 Web API
- TSP 暴力枚举算法（途经点 ≤ 8）
- 贪心策略处理模糊途经点
